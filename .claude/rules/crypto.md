# E2E Cryptography Rules

## Клиентская сторона (Angular)

### Хранилище ключей
- IndexedDB через Dexie.js или аналог
- Приватные ключи зашифрованы AES-256-GCM
- Ключ шифрования: PBKDF2 из passphrase пользователя (100k итераций)
- Опционально: WebAuthn для разблокировки без passphrase

### Signal Protocol
- libsignal-protocol-typescript для реализации
- X3DH handshake для установления сессии
- Double Ratchet для шифрования сообщений 1-на-1
- Sender Keys для групповых чатов

### Шифрование файлов
- AES-256-GCM для каждого файла
- Ключ файла генерируется случайно
- Ключ файла отправляется в зашифрованном сообщении (внутри Double Ratchet)
- Метаданные (имя, MIME) также шифруются

## Серверная сторона (Spring Boot)

### Key Bundle API
- Хранить только публичные ключи
- One-time prekeys: удалять после использования
- Signed prekey: ротация каждые 7 дней

### Что НЕ хранить на сервере
- Приватные ключи пользователей
- Plaintext сообщения
- Ключи шифрования сессий

### Сообщения
- Хранить `content` как зашифрованный base64
- Хранить `iv` (initialization vector) рядом с content
- Double Ratchet state хранится только на клиенте

## Ключевое правило
**Сервер никогда не видит plaintext сообщений.**
Всё шифрование и дешифрование — на клиенте.
